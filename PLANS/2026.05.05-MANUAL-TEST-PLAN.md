## §1 — Search scope pin (today's flagship)

Default scope is **global**; the user has to take a specific action to
constrain. Pin survives reload, conversation switching, and panel
close. It clears on explicit unpin OR when the sidebar's title-search
box gets typed into.

### §1.1 — Pin to a conversation

1. Open any conversation (pick one with a project, e.g. a Claude Code
   session, so the project-pin option also shows).
2. Look right of the conversation title — there's a button labeled
   **Search scope** with a chevron. Click it.
3. The dropdown shows two items: **Pin this conversation** (with the
   conversation title under it) and, if it's a CC session, **Pin this
   project** (with the project name).
4. Click **Pin this conversation**.
5. **Expected:**
   - Button label flips to **Pinned: conversation** with a filled (blue)
     background.
   - Sidebar rows for *other* conversations dim to ~60% opacity (still
     clickable, just visibly out of scope).
   - The URL gains `?pin=conv:<uuid>` (URL-encoded, so `:` shows as
     `%3A`).

### §1.2 — Scope chip in the SearchPanel

1. With the pin still set, press **Cmd+F** (or **Cmd+K**).
2. **Expected:** SearchPanel slides in from the right; below the search
   input, a blue pill chip reads **In: <Conversation Title>** with an
   **×** button.
3. Type a query that matches in *another* conversation but not this one
   (e.g. a project-specific term).
4. **Expected:** zero results inside the panel; an empty-state message
   reads `No matches for "<query>" in <Conversation Title>` with a
   visible button **Unpin and search all →**.
5. Click **Unpin and search all →**.
6. **Expected:** chip disappears; URL loses the `pin=` param; results
   start streaming in from the now-global search.

### §1.3 — Pin to a project (CC sessions only)

1. Navigate to a Claude Code session that lives under a project with
   ≥2 sessions in the sidebar.
2. **Search scope** dropdown → **Pin this project**.
3. **Expected:** other sessions in the *same* project remain
   full-opacity; sessions in *other* projects (and Claude Desktop
   conversations) are dimmed.
4. URL becomes `?pin=project:<encoded-path>`.
5. Open Cmd+F, type a query that matches across multiple projects.
6. **Expected:** only matches from sessions in the pinned project show
   up. Cmd+G wraps within them.

### §1.4 — Pin clears on title-search

1. Pin to a conversation (any).
2. Click the sidebar's **Search titles…** input and type a few
   characters.
3. **Expected:** the moment you start typing, the URL `pin=` param
   disappears and the sidebar dim is removed. Title search is global by
   construction; typing there signals "broaden."

### §1.5 — Pin survives reload

1. Pin to a conversation. Confirm URL has `?pin=conv:<uuid>`.
2. Hit **Cmd+R** (or browser reload).
3. **Expected:** the page comes back with the pin still active — chip
   in panel, dim in sidebar, button label still "Pinned: conversation."

**Pass:** every pin / unpin transition matches the URL state and the
visible UI (dim, chip, label) is consistent at every step.

## §2 — Search focus model (Cmd+G aria-live, single-Esc)

### §2.1 — Cmd+G keeps focus in the search input

1. Open Cmd+F. Type a query that returns ≥3 matches in the current
   conversation.
2. Cursor should be in the search input. Press **Cmd+G**.
3. **Expected:** the conversation pane scrolls to the next match and
   highlights it; the search input remains focused (you can keep
   typing without clicking back).
4. Press Cmd+G again, then again — counter advances ("1 of N", "2 of
   N", "3 of N").

### §2.2 — Match N of M is announced (a11y)

If you have VoiceOver on (Cmd+F5):
1. With matches visible and Cmd+G stepping through them, listen for
   "Match 1 of N", "Match 2 of N" announcements after each Cmd+G.
2. Without VoiceOver, you can verify via DevTools: inspect the
   SearchPanel; a `<div role="status" aria-live="polite"
   data-testid="search-match-aria-live">` should contain the current
   "Match N of M" text and update each Cmd+G.

### §2.3 — Enter on the active card focuses the message

1. With matches active, press **Enter** while focused in the search
   input.
2. **Expected:** the bubble corresponding to the active match gains the
   keyboard-selection ring; the SearchPanel **stays open**.

### §2.4 — Esc closes immediately, focus on the active-match message

1. With the panel open and a match active, press **Esc** **once**.
2. **Expected:** the SearchPanel slides out (aria-hidden=true on the
   `<aside aria-label="Search panel">`); the conversation pane retains
   the active-match message selected; pressing **j**/**k** (or Ctrl+N
   in Emacs mode) moves the message selection without re-opening
   anything.
3. The earlier two-step (Esc clears query, second Esc closes) is gone.
   If you see the query persisting after Esc, log it.

**Pass:** Cmd+G never yanks focus out of the input; Esc closes in one
press and lands on the message you were investigating.

## §3 — CC image broken-load fallback

Previously: when a `[Image: source: <abs-path>]` marker pointed at a
file that no longer exists on disk (Claude Code rotates `~/.claude/
image-cache/<session>/<N>.png`), the bubble showed the browser's
default broken-image glyph + filename — looks like an app bug.

### §3.1 — Friendly fallback

1. Reproduce by opening a Claude Code session whose image-cache
   directory has been pruned (or just any old session — high chance some
   `N.png` files have been rotated out).
2. **Expected:** any image whose backend `/api/cc-image?path=…` 404s
   shows a **dashed-border tile** with an **ImageOff** icon and the
   filename (e.g. `14.png`) in monospace. NOT a broken-glyph `<img>`.
3. The tile is still clickable — clicking it tries to open the lightbox
   (which itself will surface "image unavailable" appropriately).

### §3.2 — Successful loads still render

1. Open a recent conversation whose images are still on disk.
2. **Expected:** images render normally — the fallback only triggers on
   actual load errors (404 / blocked CORS / etc.).

**Pass:** missing CC images degrade gracefully; healthy ones still
render.
