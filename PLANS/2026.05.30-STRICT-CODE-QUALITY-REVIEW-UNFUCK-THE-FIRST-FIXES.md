- **Introduced by:** `2d775103 refactor(routes): extract ConversationToolbar`
- **Fix:** remove the `import { Button } from '@/components/ui/button'` line.

### REG-3 — `MessageInfo[]` vs `readonly Message[]` type mismatch at `ConversationPage.tsx:616`
- **Class:** P0 type assertion lie (one side of the boundary thinks `MessageInfo`, the other thinks `Message`)
- **Introduced by:** `d4f357e1` (the squashed decomposition commit, specifically the `useMessageNavigationRegistry` extraction)
- **Fix:** the hook only reads a small subset of message fields. Change the hook's `messages` parameter type from `readonly Message[]` to `readonly MessageInfo[]` (the narrower type the call site already passes). If the hook actually reads more than the `MessageInfo` shape provides, surface that — fix at the OTHER end instead. Pick narrowing over widening every time.

### REG-4 — 13 forbidden non-null assertions in two extracted hooks
- **Class:** P0 eslint `@typescript-eslint/no-non-null-assertion` (error tier)
- **Files:**
  - `frontend/src/hooks/useExpandAllToolsAnchor.ts` (8 occurrences around lines 90, 98, 115, …)
  - `frontend/src/hooks/useMessageNavigationRegistry.ts` (5 occurrences around line 149, …)
- **Introduced by:** `d4f357e1` (the squashed decomposition commit; the agent's deviation note acknowledged the choice)
- **Fix pattern (apply once per hook, not per assertion):**
  ```ts
  // BEFORE
  for (const [uuid, el] of messageRefs.current!.entries()) { … }
  const el = messageRefs.current!.get(anchorUuid)

  // AFTER — narrow once at the top of the effect/callback that needs the Map
  const refs = messageRefs.current
  if (!refs) return  // or early-return whatever the callback's "no work" shape is
  for (const [uuid, el] of refs.entries()) { … }
  const el = refs.get(anchorUuid)
  ```
  This is a 1-line addition per call site that collapses all `!` into a single narrowing. No runtime behavior change.

### REG-8 — Spurious `react-doctor/exhaustive-deps` / `react-doctor/no-derived-state-effect` eslint-disable lines
- **Class:** P1 eslint hygiene + canonical layer (the project uses TWO disable-comment syntaxes for two separate runners; the wrong one is being copied)
- **Files (≥ 5 known sites):**
  - `frontend/src/hooks/useConversationCopyAndExports.tsx:88`
  - `frontend/src/hooks/useScrollToHighlight.ts:173`
  - `frontend/src/hooks/useUnmountSafeTimer.ts:34`
  - `frontend/src/routes/ConversationPage.tsx:138`
  - (plus any other `eslint-disable-next-line react-doctor/...` in the codebase — grep)
- **Why broken:** ESLint and React Doctor are separate runners. `// eslint-disable-next-line react-doctor/...` is a no-op that ALSO triggers `Definition for rule 'react-doctor/...' was not found` as an eslint error. The correct syntax for React Doctor is `// oxlint-disable-next-line react-doctor/...`.
- **Fix:** for each site, either (a) replace `// eslint-disable-next-line react-doctor/...` with `// oxlint-disable-next-line react-doctor/...`, OR (b) delete the line entirely if the disable comment was prophylactic / decorative. Verify with `npm run lint` returning zero "Definition for rule ... was not found" errors.

These are RECOVERY commits caught and fixed within the same session as the decomposition itself. The branch never left local; nothing shipped publicly. **Fold the fix commits INTO `d4f357e1`** so `git log` reads as one clean decomposition, not a decomposition + cleanup chain.

Per `[[dual-session-history-rewrite]]` — another CC session may commit on this checkout while the recovery runs:

```bash
# 1. Snapshot before history rewrite
git rev-parse HEAD                                       # record SHA
git rev-parse HEAD^{tree}                                # record tree SHA
git branch backup/pre-unfuck-squash HEAD                 # backup

# 2. Verify nothing else landed since the recovery commits. Compare current
#    HEAD against the SHA you recorded after the LAST recovery commit. If
#    they differ, STOP — investigate before rebasing.

# 3. Interactive rebase with autosquash. The base is `d4f357e1^` (the commit
#    before the decomposition). Mark every recovery commit as `fixup` so its
#    diff folds into d4f357e1 with the decomposition's original message
#    unchanged:
GIT_SEQUENCE_EDITOR='sed -i "" "2,\$s/^pick/fixup/"' git rebase -i d4f357e1^

# 4. Verify the recovery diff actually landed
git diff backup/pre-unfuck-squash HEAD                  # MUST be empty
git rev-parse HEAD^{tree}                                # MUST equal pre-squash tree SHA

# 5. Re-run the full verification matrix on the squashed commit
#    (every static check + every test suite, per the matrix above).
#    Every command must exit 0. Every test green.

# 6. Drop the backup
git branch -D backup/pre-unfuck-squash
```

End state: `d4f357e1` (or its post-rebase replacement SHA) carries the decomposition AND every cleanup, as one topical commit. The recovery commits are absorbed; the message stays the original decomposition subject.
