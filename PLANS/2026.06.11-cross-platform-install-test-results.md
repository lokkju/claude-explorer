### 2026-06-11 — session 1 (Windows VM blocked at A2)

- A1 PASS: Windows 11 ARM64 Insider Preview VHDX (Build 26200, 13 GB) downloaded to ~/Downloads/.
- A2 BLOCKED: VM created via Virtualize-Windows wizard. Boot fell through to UEFI shell — `map -r` showed no FS mappings.
- Root cause 1: UTM Virtualize-Windows wizard initially created an empty 196 KB NVMe placeholder qcow2 instead of using the VHDX. Re-imported VHDX → "Removable Drive" greyed-checked (UTM bug for VHDX imports). After UEFI Boot Manager, NVMe entry didn't boot (no Windows Boot Manager / no valid EFI loader path found).
- Root cause 2: After detach + re-attach attempts, hit `Failed to get "write" lock` error. macOS TCC + `com.apple.quarantine` xattr on ~/Downloads/ blocking QEMU write access.
- Mitigations applied:
  - Cleared `com.apple.quarantine` xattr (didn't fix)
  - Installed qemu via brew (qemu-img 11.0.1) for VHDX → qcow2 conversion
  - Converted VHDX to qcow2 at `~/Documents/VM-disks/win11-arm64.qcow2` (13 GB sparse / 64 GB virtual). Conversion completed successfully.
  - User moved original VHDX out of ~/Downloads to `~/UTM-disks/win11-arm64.vhdx` (TCC-safer location).
- **Session ended before retry with qcow2 + new location was completed.** Resume by swapping NVMe drive in UTM Settings to point to `~/Documents/VM-disks/win11-arm64.qcow2` and trying boot again.
- Lesson: Windows-on-ARM via UTM has multiple non-obvious gotchas (Virtualize-Windows wizard creates empty placeholder, Removable flag stuck for VHDX imports, ~/Downloads TCC restrictions on imported drives). Project memory saved.

### 2026-06-11 — session 1 (UTM snapshot finding — falsifies a Council decision)

- **Finding:** UTM 4.7.5 on Apple Silicon does NOT expose snapshot UI for ANY hardware-accelerated backend (HVF AND Apple Virtualization). Confirmed by inspecting both the right-click menu and the Virtual Machine menubar — no snapshot entries anywhere.
- **Falsifies:** Council-driven plan v2 decision to use QEMU+HVF "for snapshot support" (vs Apple Virtualization for slightly smoother GUI). Both backends turn out equivalent on snapshot support — neither has any.
- **Resolution:** both Ubuntu and Windows VMs use **UTM Clone...** (APFS clone-on-write). Phase D becomes symmetric — actually simpler than the original asymmetric design. Plan + D1/D2/D3 tasks updated.
- **Lesson:** Council reviews can have wrong premises about external-system capabilities. Verify the assumed mechanism actually exists before committing to a decision based on it. Saved as memory `feedback_council_external_premise_verify.md`.
