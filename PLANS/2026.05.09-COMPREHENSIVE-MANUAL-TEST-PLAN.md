key is "no on-disk bytes in the response body".

```bash

curl -sS -i "http://localhost:8000/api/attachments/conv1/..%2F..%2Fetc/original" | head -2
curl -sS -i "http://localhost:8000/api/attachments/conv1/file1/notreal" | head -2
```

**Pass:**
- First two: 400 or 404. Body is JSON `{"detail": ...}`. **NEVER** a
  200 with on-disk file bytes.
- Third: 400 (invalid variant).
