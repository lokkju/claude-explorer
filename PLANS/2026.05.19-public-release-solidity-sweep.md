## S1 — License / SECURITY.md / CONTRIBUTING.md sanity check

### Scope

Common public-repo expectations:

- **LICENSE**: file exists, contents match the user's stated license
  intent (verify by grep + Read).
- **SECURITY.md**: how to report security issues (private channel +
  expected response time). Public V1 repos without this leak CVE
  reports to public issues.
- **CONTRIBUTING.md**: development setup pointer (mostly already in
  `CLAUDE.md`), commit-message convention, test-run instructions.
- **README badge row sanity**: any CI / coverage / version badges
  that reference private URLs.
- **`.github/` directory**: issue templates, PR template, workflow
  files — any private-org references.

### Output

- Present: list what's already there.
- Missing: file each with minimal correct content (no LLM-generated
  legal text — link to canonical sources like
  `https://www.contributor-covenant.org/`).
- Broken: any badge / link that 404s.

### Acceptance

- `LICENSE` exists, content matches user intent (ASK if ambiguous —
  don't guess between MIT/Apache/etc.).
- `SECURITY.md` exists with a working private-disclosure channel.
- `CONTRIBUTING.md` exists with a `uv sync` quickstart + test command.
- No badges in README link to private/404 URLs.

### Risks

- **License choice is user's call.** If `LICENSE` is missing, STOP
  and ask — never guess at license.
- **Private-org leaks**: `.github/CODEOWNERS` with private GitHub
  handles, internal Slack channels in templates. Grep for them.

## S4 — Fresh-install smoke test

### Scope

Simulate a fresh contributor's path from a clean clone:

1. `git clone <local>` to a scratch directory.
2. `uv sync` — does it complete on a clean cache?
3. `cd frontend && npm install` — does it complete?
4. `uv run claude-explorer --help` — does the CLI list all
   commands documented in README?
5. `uv run claude-explorer serve --port 8766` (off-default port to
   avoid colliding with any running instance) — does the server
   boot? Does `/api/config` return 200 with the new
   `config_corrupt_reason: null`?
6. `cd frontend && npm run build` — does the production build
   succeed?
7. `cd frontend && npm test -- --run` — does the test suite pass
   from scratch?
8. `uv run pytest backend/tests` — same on backend.

This catches `.gitignore` over-exclusion (e.g. fixture files
needed for tests but not committed), `pyproject.toml` missing
optional deps, and stale lock-file vs source drift.

### Output

- ✅ each step + the command used.
- ❌ for any step that needs a workaround. File it.

### Acceptance

- A contributor can `git clone` → `uv sync` → `npm install` → run
  both test suites green WITHOUT manual file fixes, env-var
  exports beyond what README documents, or asking the user.

### Risks

- **WeasyPrint / DYLD bootstrap**: macOS-specific path. Verify the
  conftest bootstrap actually sets `DYLD_FALLBACK_LIBRARY_PATH`
  before WeasyPrint import on a fresh shell.
- **Playwright browsers**: `npx playwright install` may be required
  but not in README. Surface if so.
- **`.claude/` private files**: any test fixture that points at
  `~/.claude/` will fail on a fresh contributor's machine. Grep
  conftest for them.

## S3 — Push to origin

### Scope

ONLY after S1, S2, S4, S5 are all green and committed.

```bash
git status                # confirm clean working tree
git log --oneline origin/main..HEAD | wc -l   # confirm commit count
git push origin main
```

### Acceptance

- Working tree clean.
- All commits since `origin/main` are intentional (no debug noise,
  no `wip:` commits, no AI attribution lines — grep for them).
- Push succeeds (no force-push needed — fast-forward only).

### Risks

- **Force-push**: explicitly forbidden by global CLAUDE.md. If a
  fast-forward push fails, STOP and ask. Never `--force-with-lease`
  without explicit direction.
- **AI attribution leak**: grep commits for `Co-Authored-By:
  Claude` or `🤖 Generated with` BEFORE pushing. If found, the
  user must decide whether to rebase/clean or live with it.
- **Untested commits**: if S5 surfaces HIGH findings AFTER S2
  and S4 ran, re-run S4 BEFORE pushing.
