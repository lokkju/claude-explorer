# Plan: Convert all Playwright tests to mock data (public-repo prep)

## Context

The repo can't be made public until the e2e suite passes for anyone, on any machine, without their `~/.claude-exporter/` directory. Today the suite has three coexisting modes:

1. **`mockBackend` from `e2e/fixtures.ts`** — 35 specs use route-interception. Tests inject TS-built `ConversationSummary` / `ConversationDetail` objects; routes intercepted at the browser. Does NOT depend on a live backend… mostly. The current `mockBackend` only covers `/api/config`, `/api/orgs`, and `/api/conversations**` — every other endpoint (`/api/preferences`, `/api/search`, `/api/cc-image`, `/api/bookmarks`, `/api/files`, `/api/fetch`, `/api/export`, etc.) leaks through to whichever backend is running.
2. **Raw `@playwright/test` + `waitForConnection`** — 9 specs (`conversations`, `copy-button-label`, `dark-mode`, `jump-buttons`, `mobile-drawer`, `search`, `settings-esc`, `settings`, `theme`) have NO mocks. They call `waitForConnection()` then assert against whatever the backend serves. Today `playwright.config.ts` boots a backend pointed at `tests/fixtures/desktop` + `tests/fixtures/claude` (synthetic JSONs checked into the repo) — but only when `PLAYWRIGHT_LIVE_DATA` is unset.
3. **`PLAYWRIGHT_LIVE_DATA=1` mode** — boots backend with `CLAUDE_EXPORTER_DATA_DIR` unset, so the user's real `~/.claude-exporter/` data drives the suite. Used today because the dev backend is already on port 8000. Causes ~30 failures because tests baked in assumptions about default UI state (`rightPaneTab=search`, `keyboardMode=emacs`) that the user's real `preferences.json` overrides post-P3 migrations.

Goal: a single mock-only mode. `npm run test:e2e` runs against `vite` only — no Python backend at all. Every API call hits a mock. No path through the suite reads `~/.claude-exporter/` or `tests/fixtures/`. Repo becomes publishable.

## User decisions (from AskUserQuestion 2026-05-06)

- **Q1: PLAYWRIGHT_LIVE_DATA fate** → **Remove entirely.**
- **Q2: `tests/fixtures/{desktop,claude}` JSON fixtures** → **Keep, but move under `backend/tests/fixtures/` for backend pytest use only.** Frontend specs no longer touch them.
- **Q3: Rollout** → **Phased, many small commits.**
- **Q4: Specs that need real backend** → **Audit case-by-case during conversion.** Surface any that genuinely require live behavior; decide fate at that point. Default expectation: 0–3 such specs.

- `playwright.config.ts:21` switches modes via `USE_FIXTURES = !process.env.PLAYWRIGHT_LIVE_DATA`. Boots backend with env vars in fixture mode; lets backend default in live mode.

## Goal state

- Zero spec depends on a live backend.
- `mockBackend` covers EVERY backend route the app may call, with sensible defaults that can be overridden per-test.
- All test data lives in TS code (`makeSummary`, `makeMessage`, `makeDetail`). No JSON-on-disk fixtures referenced by frontend specs.
- `playwright.config.ts` boots ONLY Vite. No Python backend, no env vars pointing at fixture dirs.
- `PLAYWRIGHT_LIVE_DATA` env var, the `:live` npm script, and `waitForConnection()` are deleted (or `waitForConnection` becomes a no-op-deprecated shim during the migration).
- `tests/fixtures/{desktop,claude}/` move under `backend/tests/fixtures/` for backend pytest only.

## Phases

Each phase is one or more failing-test-first commits (where applicable). Order is smallest-first; each commit is bisectable. Local commits only; no push without explicit approval.

### Phase 1 — Expand `mockBackend` to cover all backend routes

Audit every endpoint the frontend calls (grep `apiClient`, `fetch(` in `frontend/src`). Add default route handlers in `e2e/fixtures.ts::mockBackend` for each:

- `/api/preferences` (GET → `{version: 1, data: {}}`; PATCH → echo). Critical: post-P3 every spec needs this or live prefs leak through.
- `/api/search**` → `[]`
- `/api/cc-image**` → `404 application/json {detail: "no cache"}`
- `/api/bookmarks**` → `[]`
- `/api/files/**` (Desktop attachments) → `404`
- `/api/attachments/**` (post-P4c proxy) → `404`
- `/api/fetch/**` (SSE: `start`, `complete`, immediate close) → empty stream
- `/api/export/**` → `200 application/octet-stream` empty bytes
- `/api/orgs` (already covered, verify)
- `/api/conversations**` (already covered, verify)
- `/api/health`, `/api/version`, any others surfaced by the audit.

Add `extraRoutes` parameter (already exists) to let individual tests override defaults. Test pattern: every existing `mockBackend({...})` call should keep working without changes.

**Deliverables:**
- `frontend/e2e/fixtures.ts` — extend `mockBackend` body.
- `frontend/e2e/fixtures.test.ts` — small smoke test that asserts default routes return the expected stubs.

**Commit:** `test(fixtures): mockBackend covers all backend routes by default`

### Phase 2 — Convert the 9 "naked" specs

For each of the 9 specs that import from `@playwright/test` (not `./fixtures`):

- Switch import to `./fixtures`.
- Add `mockBackend({...})` call at the start of each test, injecting TS-built fixtures (`makeSummary`, `makeDetail`) that mirror what the spec needs. Many specs only need 1–3 sample conversations.
- Remove `waitForConnection(page)` calls (or replace with a no-op).
- Replace fixture-name assertions (`"Phase 5 fixture: TLS handshakes (long)"`) with whatever the test actually asserts on; many specs just need ANY conversation to appear in the sidebar.
- Tests that explicitly rely on tokens like `NEEDLE_CC` for search assertions: build a TS fixture conversation whose message text contains the same token, plus a `mockSearch` route that returns it.

## Risks

- **Phase 2 scope is large** (9 specs, each with its own quirks). Mitigation: one-commit-per-spec keeps regressions bisectable. Each commit's CI pass is its own gate.
- **Phase 3 audit may surface a spec that genuinely needs live backend.** User's chosen path: surface it; decide then. Most likely 0 cases, but `refresh-pipeline.spec.ts` is the most plausible candidate (SSE state machine).
- **Phase 4 fixture move** could break a backend pytest if any uses a hard-coded `tests/fixtures/...` path string. Mitigation: grep first, update all references in the same commit, verify pytest is green.
- **Phase 6 `webServer` change** — if any spec implicitly relies on the backend being up (e.g. proxying through Vite to `/api`), it will fail loud. Phase 1–3 should eliminate all such dependencies; Phase 6 is the verification.
- **Snapshot tests.** If any spec uses Playwright's `toHaveScreenshot()`, the snapshots may diverge between fixture-mode and mock-mode renders (different conversation lists). None spotted in the audit, but worth a `grep`.
- **Worktree clutter.** The repo currently has 4 stale agent worktrees in `.claude/worktrees/` (from earlier in this session). Should be cleaned up before public release; out of scope for this plan but flagged here.
