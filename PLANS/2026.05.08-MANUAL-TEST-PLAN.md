## §2 — Attachments path-traversal fix (the real bug)

The `/api/attachments/<conv>/<file>/<variant>` endpoint had a real
read-leak: the resolution of `file_dir = _attachments_root() / conv_uuid
/ file_uuid` had no validation before `is_dir()`, and the only
downstream `relative_to` check validated the FINAL chosen file, not
`file_dir`. So `conv_uuid="../../etc"` and absolute-path injection
(`Path("a") / "/abs" == Path("/abs")`) both fell through to a 200 with
arbitrary on-disk file bytes when a `<variant>.*` glob matched.

Manual smoke (all should return 4xx with no body leak):

```bash
# With the backend running on localhost:8000 (real or test mode):

# 1. dotdot in conv_uuid (URL-encoded for shell safety)
curl -sS -i "http://localhost:8000/api/attachments/..%2F..%2Fetc/file/original" \
  | head -2

# 2. dotdot in file_uuid
curl -sS -i "http://localhost:8000/api/attachments/conv1/..%2F..%2Fetc/original" \
  | head -2

# 3. invalid variant
curl -sS -i "http://localhost:8000/api/attachments/conv1/file1/notreal" | head -2
```

**Pass:** each returns `HTTP/1.1 400 Bad Request` (or 404 for non-existent
conv if 400 doesn't trigger first). NEVER `200 OK`. The body is a JSON
`{"detail": ...}` and contains no on-disk filesystem bytes.

Pre-fix behavior (verified with `git stash` baseline): #1 and #2 could
return `200 OK` with arbitrary file content if a matching `<variant>.*`
existed under the resolved escape target.

## §3 — Atomic-write `.tmp` cleanup (the small bug)

`_write_atomic` in `backend/routers/preferences.py` and `_write_all` in
`backend/routers/bookmarks.py` left their `.tmp` file on disk if
`os.replace` raised. No data corruption (the original file is preserved
by the rename's atomicity), but the `.tmp` accumulated in
`~/.claude-exporter/` over time.

Verify the dir is clean:

```bash
ls -la ~/.claude-exporter/*.tmp* 2>/dev/null
```

**Pass:** no output (or `No such file or directory`). If you have any
`preferences.json.tmp` or `bookmarks.json.tmp` from a previous failed
write, delete them — they're orphans:

```bash
# Inspect first:
ls -la ~/.claude-exporter/preferences.json.tmp ~/.claude-exporter/bookmarks.json.tmp 2>/dev/null
# If they're old and the canonical files exist, remove the orphans:
rm -f ~/.claude-exporter/preferences.json.tmp ~/.claude-exporter/bookmarks.json.tmp
```

To exercise the fix end-to-end (optional; requires PATCH'ing prefs while
filesystem is read-only — easier to trust the unit test):

## §4 — SSE wire format reality check

If you have a moment, `curl` the streaming endpoints to confirm the
wire format matches the test assertions (`data:` frames only, no
`event:` headers, `: ping` keep-alives skipped):

```bash
# Need backend running with valid creds for /fetch/refresh; for
# /fetch/start use a known small org so the stream completes quickly.
curl -N -sS http://localhost:8000/api/fetch/refresh?incremental=true | head -20
```

**Pass:** every emitted line is either:

- `data: {"type": "...", ...}` — a JSON event,
- `: ping` — an SSE-comment keep-alive (capture phase only), or
- a blank line (frame separator).

There should be NO `event: <name>` lines. The test parser
(`collect_sse_data_events` in `backend/tests/conftest.py`) skips `:`
lines and discriminates by `payload["type"]`.
