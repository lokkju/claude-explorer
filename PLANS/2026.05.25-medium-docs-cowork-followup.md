## Manual UAT (still pending — see ratified plan §"Verification")

The Cowork ingestion code is shipped + tested, but **the user has not
yet run end-to-end against their real 42-Cowork-session corpus.** The
restore-script work was a side quest discovered when investigating
why Cowork files were missing from disk in the first place. Until the
user runs `sudo ./utils/restore-deleted-sessions-and-images.sh` to
restore the Cowork files Claude Desktop rotated off disk, and then
restarts `claude-explorer serve`, the Phase-1-through-7 work is
unconfirmed against real data.

UAT checklist (from ratified plan):

1. `sudo ./utils/restore-deleted-sessions-and-images.sh --dry-run` →
   confirm Cowork files appear in the dry-run plan.
2. `sudo ./utils/restore-deleted-sessions-and-images.sh` → actually
   restore.
3. Restart `claude-explorer serve`. Watch lifespan log for "Loaded N
   Cowork sessions from ~/Library/Application Support/Claude/...".
4. Sidebar → select "Claude Cowork" → expect ~42 sessions with
   sensible titles.
5. Search "Marc" → expect at least one Cowork hit; click → conversation
   pane opens with messages.
6. Export one Cowork session to Markdown → content survives, no
   `_audit_hmac` leak.
7. Open Claude Desktop, archive one Cowork session, refresh Explorer →
   it disappears. Toggle "Show archived" → it reappears.
8. Confirm the error-field banner (D9) renders if any Cowork session
   has a non-null `error`.
