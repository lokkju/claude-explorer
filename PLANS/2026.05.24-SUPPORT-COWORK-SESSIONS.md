## Context

Claude Desktop's "Cowork" tab is a Claude Code–style local agent for non-coding work — same harness, but sessions live entirely on disk under `~/Library/Application Support/Claude/local-agent-mode-sessions/<deployment>/<org>/`, never served by any API. The user reported "Cowork messages are missing"; the multi-org fetch path was a red herring. ~42 sessions / 134 MB / 5 months of the user's data are wholly invisible to Claude Explorer today.

Outcome: Cowork sessions surface in the sidebar, are searchable, exportable, and filter-isolatable via the existing source selector — read-only, never writing to Claude Desktop's data dir. The Cowork `audit.jsonl` is the *same* JSONL shape the Claude Code reader already parses, so the work is mostly **reusing the 1400-LOC CC parser** plus **wiring a third `source` value through the stack**.

Scope confirmed with maintainer: **all phases** (core ingestion + UX polish + Time Machine restore-script extension + Medium docs). Path resolution: **add `platformdirs`** as a dependency.

## Corrections to the original draft (verified against the code)

1. **DROP the `thinking` schema change.** `backend/models.py:12` already types it as open `type: str`, and `frontend/src/lib/types.ts:120-129` *explicitly forbids* tightening it to a `Literal`. `thinking` blocks already round-trip and are silently dropped at render (`ContentBlockRenderer.tsx:106` `default: return null`) for **both** CC and Desktop today. Cowork inherits identical behavior for free. No schema change.

2. **Source-literal widening is ~22 sites, not 2.** Every `Literal["all","CLAUDE_AI","CLAUDE_CODE"]` / `Literal["CLAUDE_AI","CLAUDE_CODE"]` must gain `"CLAUDE_COWORK"`: `models.py` (×2: 120, 174), `search.py` (×5), `store.py` (×4), `search_index.py` (×8), `routers/conversations.py` (×1), `routers/search.py` (×2). Plus the frontend `SourceFilter` union + `SOURCE_FILTERS` array (`types.ts:332,344`).

3. **Detail-route resolver needs a Cowork branch — the draft omits this.** `store.py:_find_conversation_data` (≈line 629) resolves a clicked uuid → data for the detail/export endpoints. It only walks Desktop JSON + CC JSONL. Without a Cowork branch, clicking a Cowork row 404s. **Must-have.**

4. **Search-index dispatch collides on `.jsonl`.** `search_index.py:_load_conversation_at` (≈1769) dispatches purely on `path.suffix == ".jsonl"` → CC reader. Cowork's messages file is *also* `audit.jsonl`, so it'd be wrongly parsed by the CC reader. Fix: `_enumerate_conversation_paths` (≈1726) already returns `(path, source)` tuples — tag Cowork paths `"CLAUDE_COWORK"` and make `_load_conversation_at` dispatch on the **source tag** (thread it through), not the extension. Also `_resolve_path_for_conv` (≈1698) needs a Cowork branch for drift detection.

5. **`normalize_session_fields` must map MORE than `session_id`.** The shared transforms read `type`, `message.content`, `uuid`, `parentUuid`, `timestamp`, `sessionId`, `message.id`, `cwd`/`gitBranch`/`version`, `message.model`. The draft's helper only maps `session_id`→`sessionId` and **strips** `_audit_timestamp`. If Cowork lines carry `_audit_timestamp` but not `timestamp`, stripping makes every message fall back to `datetime.now()`. If lines lack `uuid`/`parentUuid`, `_get_message_key` collapses **all** user messages into one. → Map `_audit_timestamp`→`timestamp` (when `timestamp` absent); synthesize a stable `uuid` per line if absent; verify `parentUuid`. **See Phase 0a.**

6. **Restore script has no `RELATIVE_PATHS` array.** `utils/restore-deleted-sessions-and-images.sh` uses `DOMAINS=("projects" "image-cache")` and `resolve_domain_base_in_backup()` hardwires the `.claude/$domain` suffix (lines 459-465). Cowork lives under a **different root** (`Library/Application Support/Claude/...`). Needs a per-domain base-path mapping + an allow-list filter in the index walk. **See Phase 7.**

**Stay CC-specific** (do NOT move): `_is_system_message`, `_title_from_entry`/`_TITLE_FIELD_BY_TYPE`, `read_conversation_summary_fast`, `_extract_conversation_metadata`, slash-command collapse / prelude / canned-fold passes.

```python
# backend/agent_session_io.py
def normalize_session_fields(entry: dict, *, fmt: Literal["claude_code", "cowork"]) -> dict:
    """Normalize a parsed JSONL line to the CC field shape the shared transforms expect.
    No-op for claude_code. For cowork: map session_id→sessionId, map _audit_timestamp→
    timestamp (only if timestamp absent), drop _audit_hmac. Exact map finalized in Phase 0a."""
    if fmt == "claude_code":
        return entry
    out = dict(entry)
    if "sessionId" not in out and "session_id" in out:
        out["sessionId"] = out["session_id"]
    if "timestamp" not in out and "_audit_timestamp" in out:
        out["timestamp"] = out["_audit_timestamp"]
    out.pop("_audit_hmac", None)
    # uuid synthesis / parentUuid handling per Phase 0a findings.
    return out
```

**Hard gate:** full existing CC suite (`test_cc_*.py`, `test_claude_code_title.py`, `test_cc_module_split.py`) must pass **unchanged**. Fallback if the physical move is churny: `cowork_reader.py` imports the pure functions directly from `cc_message_transforms` — `agent_session_io` is organizational, not load-bearing.
