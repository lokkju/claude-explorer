### T2 — 4 Medium S5 findings

Each is its own RED→GREEN→commit cycle.

**T2a — `frontend/src/lib/api.ts` runtime-validate search response**
- Sites: `api.ts:159, 185` use `res.json() as Promise<SearchResponse>`
  which is a runtime lie — TypeScript believes the cast, but a backend
  shape change goes undetected until a downstream consumer crashes.
- Fix: introduce a `parseSearchResponse(unknown): SearchResponse`
  validator. Hand-rolled (no Zod dependency added). On shape mismatch,
  throw an `ApiError` with a descriptive message.
- Tests: vitest pair — valid envelope passes, malformed envelope (drop
  `results`, drop `total_messages_matched`, swap a field type) throws.

**T2b — Thread `AbortSignal` through 6 queryFn**
- Sites: `getOrgs`, `getConfig`, `getConfigStats`, `getConversationTree`,
  `fetchPrefs`, plus any other queryFn without `signal`.
- Fix: each api.ts method gets an optional `signal?: AbortSignal` param
  passed through to `fetch(..., { signal })`. Each hook's `queryFn`
  forwards the TanStack `signal`.
- Tests: vitest that aborts mid-request and asserts the fetch was
  cancelled (rejected with `AbortError`). One representative test
  covers the pattern; the others get a smoke test (signal is wired).

**T2c — `backend/cache.py` `get_or_load + set` TOCTOU**
- Site: `get_or_load` returns cached content then `set` writes the new
  payload — a concurrent writer can interleave between the read and
  the set, and the cache stores stale content.
- Fix: check-read-check (stat before, stat after; on mismatch, skip the
  set and re-load). Same shape as the prior `search_index.py` fix.
- Tests: `threading.Barrier` pair that exercises the race and asserts
  the cache holds the post-race content, not the pre-race content.

**T2d — `frontend/src/components/conversation/ConversationPage.tsx` setTimeout cleanup**
- Sites: 4 unguarded `setTimeout` calls (copy-feedback timers, scroll-
  to-match timers).
- Fix: each `setTimeout` returns a handle stored in a ref; the
  component's effect-cleanup calls `clearTimeout`. If the component
  unmounts mid-timer, no setState fires on a dead component.
- Tests: vitest that unmounts the component within the timer window
  and asserts no warning is logged. (The 6-context-provider mocking
  gap S5 flagged is sidestepped by rendering the specific subcomponent
  that owns the timer, not the whole page.)

### T4 — Pre-push history scan

**Run BEFORE push** (and codified in `CLAUDE.md` for future pushes):

```
# Secrets across all 187 commits
git log -p origin/main..HEAD | grep -nE 'sk-ant-|Bearer [A-Za-z0-9]|password\s*[=:]'

# Personal paths in committed files
git grep -nE '/Users/[a-zA-Z]+/|/home/[a-zA-Z]+/' -- ':!PLANS/' ':!*.md'

# Real session keys / cookies in test fixtures
git grep -nE 'sessionKey.*[a-zA-Z0-9]{20,}' -- 'backend/tests/' 'fetcher/tests/'

# AI attribution across ALL 187 commits
git log origin/main..HEAD --format='%H %B' | grep -iE 'co-authored-by:.*claude|🤖 generated'

# Accidentally-committed files
git ls-files | grep -E '^\.env$|\.env\.|credentials\.json$|\.sqlite$|\.local$'

# ~/.claude paths in files shipped to PyPI sdist
uv build --sdist && tar tzf dist/*.tar.gz | xargs -I {} sh -c '...' # see CLAUDE.md
```

Each grep MUST return zero matches (or only known-OK matches the
user has eyeballed) before push.
