## §1 — Image attachments (Phase 2 + G3a)

The biggest user-visible change. Find a Desktop conversation with screenshots
in it (e.g. the Synology one, or any conv where you pasted/uploaded an image).

1. **Inline thumbnails render.** Open the conversation. Each message that
   originally had an image attachment should now show a thumbnail tile (or
   a 2-col grid for multi-image messages).
   - **Single image** → tile is `object-contain` capped at `max-h-64`, full
     aspect ratio.
   - **2-4 images** → 2-column grid of `aspect-square object-cover` tiles.
   - **5+ images** → 4 squares + a `"+N"` overflow tile.
   - **Images load lazily**: scroll past several image-bearing messages
     quickly; check Network tab for `/api/.../thumbnail` requests staggered,
     not all at once.

2. **Lightbox on click.**
   - Click any thumbnail → full-screen dark overlay opens with the image
     fitted to viewport (`object-contain`).
   - Header shows: **filename · counter (i/N)** · **Download** icon ·
     **Open in new tab** icon · **Close** (X).
   - **Esc** closes (Radix native).
   - On a multi-image message: **←** / **→** navigate (wraps both
     directions); counter updates live.
   - Press **`d`** → triggers a download of the file.
   - Press **`o`** → opens the original in a new tab.
   - Click outside the image (on the dark scrim) → closes (Radix native).

3. **Tool toggle independence.** Click `Tools` in the conversation header to
   turn off tool blocks. Image attachments should **stay rendered** (Council
   Q7).

4. **Markdown export contains image refs.** Click the **Markdown** export
   button. Save the .md file. Open it in a viewer:
   - **VS Code preview** (with backend still running on :8000) → images
     render inline.
   - **`cat` the file** → expect lines like
     `![Image attachment: foo.png](/api/.../preview)` after each message
     that had an attachment.
   - Copy/paste into a serverless context (Medium draft, GitHub Gist) →
     image refs read as link text since claude.ai-relative URLs don't
     resolve. Article calls this out.

5. **Copy as Markdown.** Header button → paste into a text editor → image
   refs present in the clipboard payload.

6. **PDF export with images.** Click the **PDF** export button. Open the
   resulting `.pdf`:
   - Image attachments render as `<figure>` blocks fitted to page width
     (max-h: 480px so landscape doesn't blow out the page).
   - Filename appears as a small caption under each image.
   - Tool toggle still affects tool blocks but **never** affects images.

7. **Broken image fallback.** Hard to test live — would require an image
   whose CDN URL returns 404. If you don't see one, skip. (Backend tests
   cover the placeholder rendering.)

## §2 — Cross-conversation Cmd+G (bug fix from `848663b`)

1. Open a Desktop conversation.
2. **Cmd+K** → SearchPanel opens.
3. Type a query that has matches in **multiple** conversations (e.g. "the").
4. Press **Cmd+G** repeatedly. Each press should:
   - Highlight the next match card in the panel.
   - **Navigate the URL** to that match's conversation (you'll see the
     conversation pane change).
   - The "N of M" counter updates.
5. **Cmd+Shift+G** to go backward.

Pre-fix, Cmd+G only highlighted the next card without navigating.

## §3 — Branch tree (article fix)

1. Find a Desktop conversation with branches (look for the `GitBranch` icon
   next to the title in the sidebar row).
2. Open it; click **View branches** in the conversation header.
3. Tree visualization opens. **Click any leaf** that's not the currently
   displayed one.
4. The conversation pane should switch to that branch's path; URL gains
   `?leaf=<uuid>`. Hit browser back; URL pops, conversation switches back.

This is the behavior the article now describes. Pre-Build-8-#6 the tree was
just a static diagram.

## What to look for that might be broken

- **Images not rendering** because the `/api/...` proxy doesn't pass
  through Cookie auth correctly. Check `Network` tab for 401/403 on
  `/api/.../thumbnail`.
- **Lightbox arrow nav** going wrong on 0-image messages.
- **Cmd+G navigating to a conversation you didn't expect** because the
  auto-navigate fires on every `activeMatchIndex` change. That's the new
  behavior the article documents.
- **Markdown export reads as a wall of file paths** when opened standalone
  — that's the relative-URL caveat the article calls out.

Report back with anything broken; we iterate.
