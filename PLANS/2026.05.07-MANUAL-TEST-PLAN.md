## §5 — Canonical composition examples

### §5.1 — One atom hides multiple patterns (the cron case)

The least-friction multi-pattern hide is a single atom:

1. New atom. Behavior: **Hide matches**. Mode: Glob.
2. Patterns:
   ```
   cron1
   cron2
   ```
3. Save. Activate.
4. **Pass:** every conversation whose title matches `cron1` OR `cron2` (glob substring match) is hidden.

### §5.2 — Two hide-atoms ANDed via group (alternative composition)

For the same effect via a group:

1. Atom A: Behavior=Hide, Patterns=`cron1`.
2. Atom B: Behavior=Hide, Patterns=`cron2`.
3. Group G: Match=`all of these`, Members=[A, B].
4. Activate G.
5. **Pass:** rows where A AND B both keep (i.e., neither cron1 nor cron2 matches) are visible. Rows that match either pattern are hidden by their respective atoms' Hide behavior.

The atom shortcut (§5.1) is simpler; the group form is necessary only if you want to enable/disable A or B independently.
