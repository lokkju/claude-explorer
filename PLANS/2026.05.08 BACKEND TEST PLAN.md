# Refined Plan: Backend test enhancement (frontend-spec-driven gap closure)

## Context

The existing plan (`PLANS/2026.05.07-backend-test-enhancement.md`) closes
five systemic gaps in the ~155 backend tests against a frontend-derived
API contract: SSE coverage, concurrency, security-adjacent input,
migration legacy seeds, and weak status-only assertions. The council
(Gemini-3-Pro + GPT-5.3) signed off with edits on 2026-05-07.

Reading the actual code surfaced four corrections to the original plan.
This refinement folds them in, trims the council deliberation prose
(decisions retained), and adds concrete file:line targets so the
implementer can act without follow-ups.

### Code-vs-plan corrections (load-bearing)

1. **SSE wire format.** `backend/routers/fetch.py:194-195` and
   `:478-479` show `_send_event` emits ONLY `data: {json}\n\n`. There
   are no `event: <name>` lines. The original plan's
   `collect_sse_events` helper that parses `event:` headers, and the
   "grep for `event: start|progress|complete|capture_*`" rationale,
   are wrong against this impl. Tests must parse `data:` frames and
   discriminate by `payload["type"]`. The capture phase emits
   `: ping\n\n` SSE-comment keep-alives in addition to `data:`
   frames (`fetch.py:519`, `:932-933`); the parser must skip those.

2. **The declared `_refresh_lock` is never acquired.**
   `backend/routers/fetch.py:41` declares `_refresh_lock: asyncio.Lock`
   but `refresh_pipeline` (`fetch.py:1085-1108`) only does a
   read-then-write of `_refresh_in_progress` with no `async with`
   around the lock. In single-threaded asyncio there's no `await`
   between the check and the set, so two concurrent requests are
   still serialized correctly — but the test should target the
   actual mechanism (the flag), not an idealized lock. Optional
   follow-up: either remove the unused lock or wrap the flag in it.

3. **Permission bits are already explicit.** `preferences.py:71`
   chmods `tmp` to `0o600` BEFORE `os.replace` (mode survives rename),
   and `fetcher/credentials.py:303` does the same for credentials.
   Gemini's "heavy bet that the temp file inherits 0o644" is likely
   wrong against this impl. The P2.7 test still ships as
   regression-prevention, framed as "lock in the contract" rather
   than "expected bug".

4. **`/api/attachments` traversal has a real gap.** `files.py:291` does
   `_attachments_root() / conv_uuid / file_uuid` with no validation
   on the path segments before `is_dir()`. The
   `chosen.resolve().relative_to(file_dir.resolve())` check on
   `:309` only validates the final chosen file, not whether
   `file_dir` itself escaped. A `conv_uuid="../../etc"` directory
   probe would currently fall through to a 404 (no such dir under
   the resolved escape target), but the contract should be 400 with
   no fs touch. P2.6 should expect failure on first run and ship a
   small impl fix alongside the test (a `relative_to` check on
   `file_dir` after `resolve()`).

### P0 · Conftest helpers (foundation; lands first)

Edit `backend/tests/conftest.py`. Add:

- `isolated_data_dir(tmp_path, monkeypatch)` — creates
  `<tmp_path>/data` (a SUBDIRECTORY of tmp_path, not tmp_path
  itself), sets `CLAUDE_EXPORTER_DATA_DIR=<tmp_path>/data`, calls
  `backend.config.get_settings.cache_clear()` before yield and on
  teardown. The subdirectory matters because `_resolve_path` uses
  `data_dir.parent / "preferences.json"` — see also `legacy_v1_prefs`
  below and the matching Risk note. Per CLAUDE-TESTING.md §5.1.
- `real_async_client(app)` — yields
  `httpx.AsyncClient(transport=ASGITransport(app=app),
  base_url="http://test")`. Use for SSE / concurrency tests.
- `collect_sse_data_events(resp, *, stop_on=("complete","error"),
  timeout=5.0)` — async generator that consumes `aiter_lines`,
  **skips lines starting with `:` (SSE comments)**, parses `data:`
  frames, json-decodes the payload, yields `(payload["type"],
  payload)` tuples until a `stop_on` type arrives or the deadline
  fires. Wraps an `asyncio.wait_for` so a hung stream fails fast.
  This replaces the original plan's `collect_sse_events`; the
  rename is intentional to flag the wire-format reality.
- `legacy_v1_prefs(isolated_data_dir)` — writes a known v1
  `preferences.json` shape (with `polarity`, `pinned`,
  `activeFilterIds`, no `_migratedV2`) under
  `<isolated_data_dir>/../preferences.json`. Used by P2.8/P2.9.
- `_isolated_credentials_path(tmp_path, monkeypatch)` — monkeypatches
  `fetcher.credentials.DEFAULT_CREDENTIALS_PATH`,
  `fetcher.bulk_fetch.DEFAULT_CREDENTIALS_PATH`, AND
  `backend.routers.fetch.DEFAULT_CREDENTIALS_PATH` to a tmp_path.
  THREE bindings, not two — `bulk_fetch.py` also imports the constant
  by value at module load. Confirmed by BT-P0 implementation; the
  original plan said two. Plus pin `CLAUDE_DIR` env var so any code
  paths that resolve `~/.claude-exporter/` from scratch don't bleed
  into the user's real dir.
- `reset_refresh_flag()` autouse fixture — sets
  `backend.routers.fetch._refresh_in_progress = False` before each
  test so a leaked flag from one test doesn't 409 the next.

## Risks

- **SSE-format reality.** The plan now targets `data:`-only frames
  with a `type` discriminator. If a future router migrates to
  named-event SSE (`event: foo\ndata: {…}`), tests will need to
  follow; flag in P1.2 gap-matrix banner.
- **Concurrency test flakiness.** The flag check is synchronous in
  `refresh_pipeline` (no `await` between read and write), so
  `asyncio.gather` is sufficient — but the moment the impl gains
  any `await` between the check and set, the test must wrap with an
  `asyncio.Event` barrier inside the streamer to remain
  deterministic.
- **Path-traversal tests must not actually leak.** Assert REJECT +
