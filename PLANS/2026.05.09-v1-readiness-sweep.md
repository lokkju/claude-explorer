## Phase A — Issue 3: `_proxy()` local-cache fallback (broken images) — TIER 1

**Goal:** when claude.ai upstream returns 404 for a previously-cached
image, serve the local cached copy. Bonus: fixes Markdown/PDF exports
(same proxy URL).

### Files
- `backend/routers/files.py` (~12 lines added in `_proxy()` upstream-404
  branch + 1 log line)
- `backend/tests/test_files_proxy.py` (NEW — 3 tests, ~90 lines)

### Implementation sketch
- In `_proxy()`, when `upstream.status_code == 404`:
  ```python
  matches = [
      m for m in _attachments_root().glob(f"*/{file_uuid}/{variant}.*")
      if m.is_file()  # guard against directories matching the pattern (Gemini)
  ]
  if matches:
      logger.info(
          "proxy_local_fallback",
          extra={"file_uuid": file_uuid, "variant": variant, "path": str(matches[0])},
      )
      media_type = mimetypes.guess_type(str(matches[0]))[0] or "application/octet-stream"
      return FileResponse(matches[0], media_type=media_type, headers=headers)
  raise HTTPException(status_code=404, detail="image not found upstream and no local cache")
  ```

## Phase B — Issue 1: Connection dialog UX (3 bundled fixes)

**Goal:** stop the contradictory "Last error" line during reconnect,
give the backend cold-start time before flashing the dialog, and
extend the first-retry interval so transient 1-second blips don't
trigger a dialog at all. **All three sub-fixes ship together** per V1
polish priority.

### Files
- `frontend/src/components/ConnectionStatus.tsx` (3 surgical changes:
  line 181 gate, line ~69 retry-schedule, line ~136 dialog-open
  threshold)
- `frontend/e2e/connection-status.spec.ts` (NEW or extend — 3
  regression tests)

### Implementation sketches
1. **Hide stale `lastError` while connecting** — `ConnectionStatus.tsx:181`:
   ```tsx
   {lastError && state !== 'connecting' && (
     <p className="text-xs text-red-500">Last error: {lastError}</p>
   )}
   ```
2. **Bump first-retry from 2s to 4s** — `ConnectionStatus.tsx:69`:
   ```ts
   const delay = Math.min(2000 * Math.pow(2, attempt), 10_000)
   // (was: 1000 * Math.pow(2, attempt))
   ```
   Backend `--reload` cold-start is 5–10s; the old 2s first-retry
   guaranteed a failure inside the boot window. New schedule:
   `4 → 8 → 10 → 10 → 10 = 42s` to give up; first user-visible failure
   is now **after** boot completes for a healthy backend.
3. **Suppress dialog until retry ≥ 2** — `ConnectionStatus.tsx:136`:
   ```tsx
   const showDialog = retryCount >= 2
   // (was: retryCount > 0)
   ```
   A one-off transient (Wi-Fi blip, single dropped request) never
   flashes the dialog.

### Tests (one per sub-fix)
1. **`connection-status — no stale error during connecting`**:
   force `state = 'connecting'` via mock backend that hangs
   `/api/config`. Assert `getByText(/Attempting to connect/)` visible
   AND `getByText(/Last error/)` NOT visible.
2. **`connection-status — first retry waits 4s`**:
   spy on the timeout schedule; assert first delay is 4000ms (use
   Playwright clock-control if needed, or just inject a test hook).
3. **`connection-status — single transient does not show dialog`**:
   mock backend that fails once then succeeds. Assert dialog never
   becomes visible. (`retryCount` reaches 1 then resets.)

### Bidirectional verification (per sub-fix)
Revert each change individually; run the matching test; confirm fail.

### Risk: state transition exists?
Verify `state` actually transitions OUT of `'connecting'` on permanent
failure (otherwise the new gate would hide errors forever). Read the
state machine before C2; if it doesn't transition, the gate condition
needs to be `(state === 'failed' || state === 'idle')` instead of
`state !== 'connecting'`.

### Commit
`fix(connection-status): no stale error during reconnect; 4s first retry; dialog after retry≥2 (V1 polish)`

(Single commit — three small surgical changes to one file, one test file.)

## Phase D — §6A: Fix WeasyPrint library loading (4 PDF tests)

**Goal:** all 4 PDF tests pass without `--ignore` flags. Manual test
plan stops needing the WeasyPrint env-var workaround.

### The actual problem (verified just now)
- `OSError: cannot load library 'libgobject-2.0-0'`
- Library IS installed at `/opt/homebrew/lib/libgobject-2.0.0.dylib`
  (note `.0` not `-0`).
- WeasyPrint's CFFI binding asks for `libgobject-2.0-0` (dash) which
  isn't a symlink Homebrew creates.
- `DYLD_FALLBACK_LIBRARY_PATH` doesn't help: macOS SIP strips
  `DYLD_*` env vars from subprocess invocations (pytest → uv → python).

### Fix: developer-setup symlink script + auto-skip safety net

**D1 — `scripts/dev/symlink-weasyprint-libs.sh`** (NEW):
```bash
#!/usr/bin/env bash
# Create dash-form symlinks WeasyPrint's CFFI bindings expect on macOS.
# Run once after `brew install pango cairo libffi`.
set -euo pipefail
LIB_DIR="$(brew --prefix)/lib"  # arch-agnostic: /opt/homebrew on Apple Silicon, /usr/local on Intel
declare -A LINKS=(
  ["libgobject-2.0-0"]="libgobject-2.0.0.dylib"
  ["libpango-1.0-0"]="libpango-1.0.0.dylib"
  ["libpangocairo-1.0-0"]="libpangocairo-1.0.0.dylib"
  ["libpangoft2-1.0-0"]="libpangoft2-1.0.0.dylib"
  ["libcairo-2"]="libcairo.2.dylib"
  ["libgio-2.0-0"]="libgio-2.0.0.dylib"
  ["libfontconfig-1"]="libfontconfig.1.dylib"
  ["libharfbuzz-0"]="libharfbuzz.0.dylib"
)
for link in "${!LINKS[@]}"; do
  target="${LIB_DIR}/${LINKS[$link]}"
  if [[ ! -e "$target" ]]; then
    echo "WARN: $target not found — skipping $link"
    continue
  fi
  ln -sf "$target" "${LIB_DIR}/${link}"
  echo "linked ${LIB_DIR}/${link} -> ${LINKS[$link]}"
done
echo "Done. Run: uv run pytest backend/tests/test_export_pdf_images.py -v"
```

Mark executable; document in `CLAUDE.md` "PDF Export Dependencies"
section.

**D2 — `backend/tests/conftest.py`**: add an autouse `pytest.skip`
for PDF tests when WeasyPrint can't import:
```python
def _weasyprint_available() -> bool:
    try:
        import weasyprint  # noqa: F401
        return True
    except (ImportError, OSError):
        return False

@pytest.fixture(autouse=True)
def _skip_if_pdf_test_and_no_weasyprint(request):
    """Skip PDF-export tests with a clear message instead of OSError when
    WeasyPrint native libs (libgobject, libpango, etc.) aren't loadable.
    See scripts/dev/symlink-weasyprint-libs.sh on macOS."""
    if "pdf" not in request.node.nodeid.lower():

    if not _weasyprint_available():
        pytest.skip(
            "WeasyPrint native libs not loadable. "
            "On macOS run: bash scripts/dev/symlink-weasyprint-libs.sh"
        )
```

Net effect: if dev hasn't run the symlink script, the 4 tests show as
`SKIPPED` with a clear actionable message instead of `FAILED` with a
cryptic CFFI traceback. After running the script, they show as
`PASSED`.

## Phase E — §6B: Fix the 3 Playwright flakes

### E1 — `markdown-bundle-export.spec.ts:33` — Settings persistence parallel state-bleed

**Diagnosis:** `usePreferences` PATCHes `/api/preferences` after
`check()`. Under parallel workers, `mockBackend`'s preferences blob is
shared per-worker; race between PATCH commit and reload-time GET.

**Fix:** force serial mode for this describe block. The test is a
single legacy-Settings smoke; serializing it has zero blast radius.
```ts
test.describe.configure({ mode: 'serial' })
test.describe('Markdown bundle Settings persistence (Issue #4 legacy controls)', () => {
  // ...
})
```

Alternatively, wait for the PATCH to commit AND let React settle the
checked-state before reloading (Gemini edit — the second `expect`
ensures React state has caught up with the network round-trip):
```ts
const patchPromise = page.waitForResponse(r =>
  r.url().endsWith('/api/preferences') && r.request().method() === 'PATCH'
)
await page.getByLabel('Obsidian').check()
await patchPromise
await expect(page.getByLabel('Obsidian')).toBeChecked()  // settle React
await page.reload()
```

**Recommended:** the explicit `waitForResponse` + UI-settled assertion
— fixes both the network race AND the React-state race without
losing parallel-execution speed for adjacent tests.

### E2 — `sidebar-behavior.spec.ts:148` — strict-mode locator collision

**Diagnosis:** `page.getByRole('checkbox').filter({ hasText: '' }).first()`
is fragile — multiple checkboxes exist, `.first()` ordering depends on
DOM render timing.

**Fix:** replace with accessible-name lookup. Currently:
```ts
const groupToggle = page.getByRole('checkbox').filter({ hasText: '' }).first()
const groupLabel = page.locator('label[title*="Group sessions by project"]')
await groupLabel.click()
```

Replace `groupToggle` (it's unused after this line anyway — only
`groupLabel.click()` actually fires) and the locator can be tightened
to:
```ts
const groupCheckbox = page.getByRole('checkbox', { name: 'Group sessions by project' })
await groupCheckbox.check()
```

**Verify:** the actual `<label>` for that checkbox must have its
accessible name match. Read `frontend/src/components/Sidebar.tsx` (or
wherever the checkbox lives) and confirm; if the accessible name is
different, use the actual one.

### E3 — `theme.spec.ts:44` — initial-title race

**Diagnosis:** `getAttribute('title')` doesn't auto-wait. SettingsContext
hydration from localStorage may be mid-flight when `initialTitle`
reads.

**Fix:** add a hydration gate before reading initial title:
```ts
test('theme toggle button cycles through themes', async ({ page }) => {
  await page.goto('/')
  const themeButton = page.locator('button[title*="Theme:"]')
  // Hydration gate: wait for the title to match a known theme word.
  await expect(themeButton).toHaveAttribute('title', /light|dark|system/)
  const initialTitle = await themeButton.getAttribute('title')
  await themeButton.click()
  // Auto-retry the assertion instead of single-shot getAttribute.
  await expect(themeButton).not.toHaveAttribute('title', initialTitle ?? '')
})
```

Two changes: hydration gate + replacing the single-shot
`getAttribute` post-click with `await expect(...).not.toHaveAttribute(...)`
which auto-retries.

## Risks

- **Issue 3 glob performance.** ≤14 conv-dirs today; negligible.
  WWCMM at 100k+ convs: SQLite-index. Document in `CLAUDE.md` if it
  becomes a real worry.
- **Issue 3 fallback could mask upstream-degradation incidents.**
  Mitigated by the new log line ("proxy_local_fallback") so logs/Sentry
  can distinguish "we served local fallback" (HTTP 200, log line)
  from "actually gone" (HTTP 404 with new detail).
- **Issue 1 hides errors during `connecting`.** Intentional — spinner
  conveys the same info. **MUST verify** state transitions out of
  `connecting` on permanent failure before C2; if it doesn't, the
  user would never see the error. Adjust gate to
  `(state === 'failed' || state === 'idle')` if needed.
- **Issue 2 jumps could disorient users mid-typing.** The
  `!isSearching` gate prevents this. **MUST verify** the debounce in
  `useDebouncedQuery` is non-zero before shipping.
- **Phase D symlink script needs admin-free `ln -sf`.** `/opt/homebrew/lib`
  is user-writable on Apple-silicon Homebrew installs; the script
  should fail loudly if write-permission missing. Test on a fresh
  account to confirm.
- **Phase E2 (sidebar locator) requires verifying the `<label>` has
  matching accessible name.** Read `Sidebar.tsx` and confirm before
  shipping; if `aria-label` is different, use that.
- **Playwright `--repeat-each=5` may surface NEW flakes** beyond the
  3 known. If so, capture as Phase E4-Ek and decide whether to fix
  them in this V1 sweep or defer with explicit justification.
- **Backend cold-start under uvicorn `--reload` may exceed even the
  4s first-retry.** If so, Phase B sub-fix #2 needs to bump higher
  (e.g., 6s); measure cold-start time in dev and tune.
