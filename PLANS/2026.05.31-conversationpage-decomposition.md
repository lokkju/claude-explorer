# ConversationPage decomposition — finish the job

## Context

`frontend/src/routes/ConversationPage.tsx` is currently **1540 lines**. The strict-code-quality-review on 2026-05-30 flagged it as a P1 file-size finding. A first decomposition pass shipped four extractions (`ConversationToolbar`, `SourceBadge`, `useBracketCompactNav`, `useScrollToHighlight`) that took it from 1773 → 1540. The /coding agent explicitly deferred the remaining `ConversationHeader` extraction because it didn't want to do four same-file refactors in one batch.

This plan finishes the decomposition: 9 small commits, each a coherent unit, taking the file to **≈700 lines** of pure orchestration. The B principle's advisory gate is ~1000; we'll be comfortably below.

**Hard constraint** (from the structural-map agent's hazard analysis): a small set of refs and mirrors **must stay owned by `ConversationPage`** because multiple extracted units read them. Violating this would re-couple the extracted pieces back through their shared parent. The constraint is documented in §"Things that stay in `ConversationPage`" below.

**Validation:** the proposed decomposition was reviewed by a Plan subagent that pushed back on five points (order swap, splitting the keyboard hook, `scrollAreaRef` ownership, `renderBubbleRow` shape, post-toggle recenter ownership). All five pushbacks are absorbed into the order below.

## Things that stay in `ConversationPage` (do NOT extract)

- **`messageRefs` Map** + the `getSetRef` callback factory. The `useCallback([])` deps list stays empty so per-uuid setter identity is stable across renders. Multiple extracted hooks/components receive these as props/args; none owns them.
- **`scrollAreaRef`**. Read by the expand-all layout effect, the post-toggle recenter effect, `scrollToTop`/`scrollToBottom`, and the toolbar handlers. Forwarded as a prop into `ConversationMessageStream`; not owned by it.
- **`pendingRecenterRef`** + `markPendingRecenter`. Written before setState in toggle handlers, read by the post-toggle recenter effect, cleared inline.
- **`visibleMessagesRef` and `virtualizerRef` mirrors**. Dodge effect deps while keeping closures fresh.
- **The post-toggle recenter effect (`ConversationPage.tsx:588–641`)**. Coupling cost exceeds LOC win — depends on 6 refs.
- **The 5 loose UI useStates** (`isTreeOpen`, `markdownDialogOpen`, `showScrollButton`/`showTopButton`, `activeCompactIdx`, `showPrelude`). They don't co-update; reducer adds ceremony without bailout savings.

## Final step — squash to one topical commit (Option 1)

After all 9 commits are green, collapse them into a **single topical commit** so `git log` reads as one decomposition pass rather than nine micro-commits. Lose bisect granularity in exchange for a clean history.

Sequence (mandatory per `[[dual-session-history-rewrite]]` — another CC session may be committing on this checkout):

1. **Snapshot before touching history.**
   ```bash
   git rev-parse HEAD                                # record SHA of last extraction commit
   git rev-parse HEAD^{tree}                         # record tree SHA for byte-equality check
   git branch backup/pre-squash-decomposition HEAD   # backup branch as a recovery point
   ```

2. **Verify nothing else landed.** Compare current `HEAD` against the SHA you recorded right after Commit 8 finished. If they differ, the other session committed mid-window — stop, investigate, do NOT rebase.

3. **Interactive rebase with autosquash off** (we're picking + squashing manually). The base is the commit BEFORE commit 0 (the plan-persist commit). Find it:
   ```bash
   git log --oneline -n 11
   # locate the commit immediately preceding the "persist plan" commit; call its SHA <BASE>
   GIT_SEQUENCE_EDITOR='sed -i "" "2,\$s/^pick/squash/"' git rebase -i <BASE>
   ```
   That marks commits 2–10 (all post-plan commits) as `squash`, leaving commit 1 (plan-persist) as the `pick`. Git will open the commit-message editor to combine them — write a single subject + body:

   ```
   refactor(routes): decompose ConversationPage 1540→700

   Carve nine cohesive units out of frontend/src/routes/ConversationPage.tsx,
   leaving the page as a pure orchestrator. Plan persisted at
   PLANS/2026.05.31-conversationpage-decomposition.md.

   - chore(conversation): move renderBubbleRow to own module
   - refactor(ui): extract ConversationViewerScrollControls
   - refactor(hooks): extract useSearchMatchHighlighting
   - refactor(hooks): extract useConversationCopyAndExports
   - refactor(routes): extract ConversationHeader
   - refactor(hooks): extract useExpandAllToolsAnchor
   - refactor(hooks): extract useMessageNavigationRegistry
   - refactor(hooks): extract useBookmarkHotkey
   - refactor(routes): extract ConversationMessageStream
   ```

4. **Verify the tree hasn't drifted.**
   ```bash
   git rev-parse HEAD^{tree}                         # must equal the tree SHA from step 1
   git diff backup/pre-squash-decomposition          # must be empty
   ```
   If either check fails, `git reset --hard backup/pre-squash-decomposition` and investigate before re-attempting.

5. **Re-run the full verification matrix from §"Verification" above.** The squashed commit must pass the same tests as the 9-commit sequence.

6. **Drop the backup branch** once the squashed commit is verified:
   ```bash
   git branch -D backup/pre-squash-decomposition
   ```

The branch stays local. The user invokes the push.
