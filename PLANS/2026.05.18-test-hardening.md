## TDD discipline

Each task starts RED:

- **C1**: write the path-traversal tests assuming the route handles
  them correctly; if the route is broken, the tests fail and surface
  a security bug to fix before V1.
- **C2**: write the new test against the real search index; the
  initial run should fail because the old mocked test was hiding a
  real-behavior gap.
- **C3**: enable xdist locally, fix any worker-collision crashes one
  at a time.
- **C4**: write tests that USE the shared builders before extracting
  them; force the API design.
- **C5**: convert one test at a time; run that test in isolation
  before the broader suite.
- **C6**: each coverage gap is a missing test — write the test,
  watch it fail, then fix the implementation if needed.
